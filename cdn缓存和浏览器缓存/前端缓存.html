判断一个网站的性能最直观的就是看网页打开的速度。其中提高网页反应速度的一个方式就是使用缓存。 一个优秀的缓存策略可以缩短网页请求资源的距离，减少延迟，由于缓存文件可以重复利用，还可以减少带宽，降低网络负荷。 web缓存分为数据库缓存、代理服务器缓存、cdn缓存、浏览器缓存。 一、浏览器缓存 1. 浏览器缓存分为强缓存和协商缓存 2. 强缓存有expires和cache-control 3. 协商缓存有last-modified跟Etag。 4. 浏览器缓存优先级 浏览器缓存-> cache-control -> expires -> Etag -> last modified 浏览器的缓存状态是由header决定的，header的参数共有四种。 1. Cache-Control: (http 1.1) 1.max-age(s) 指定设置缓存最大的有效时间。当浏览器当服务器发送请求后，在max-age这段时间里浏览器就不会再向服务器发送请求。缓存过期时间 = 第一次访问该文件的时间 + max-age < date.now Cache-Control: max-age=2592000(30天) 设置缓存几秒钟并不准确 2.s-maxage(s) 覆盖max-age或者expires 但只适用于共享缓存(public) 3.public (可以被任何对象缓存： 浏览器，代理服务器，cdn服务器) 4.private (只能是在客户端缓存) 5.no-cache 代表不缓存过期的资源，缓存会向服务器进行有效处理确认之后再处理资源，也就是缓存之前会请求服务器，使用no-cache的目的就是为了防止从缓存中获取过期的资源。 6.no-store 不进行缓存 2.Expires (http1.0) 缓存的过期时间(决对时间)，用来指定资源到期的时间，是服务器端的具体的时间点。当客户端时间与服务器时间存在时差的，将会有问题(比较少用)。Cache-Control的优先级会更高 3.Last-modified (强缓存失效后) 服务端文件的最后修改时间，需要和cache-control共同使用，会向服务器传送max-age=0,If-Modified-Since报头，询问Last-Modified时间点之后资源是否被修改过。如果没有修改，则返回码为304，使用缓存；如果修改过，则再次去服务器请求资源，返回码和首次请求相同为200，资源为服务器最新资源。 4.ETag (强缓存失效后) 优先于last-modified,服务器首先会验证Etag 根据实体内容生成一段hash字符串，标识资源的状态，由服务端产生,需要和cache-control共同使用。浏览器加上包含这串字符串传的if-none-match,max-age=0头发到服务器，验证资源是否已经修改。 浏览器请求 第一次 浏览器请求 -> 无缓存 -> 服务器 (无协商缓存)-> (200)源服务器 + 缓存头 第二次 浏览器请求 -> 命中强缓存 -> from cache -> 无缓存 -> Etag ? -> 服务器+if-none-match + max-age=0 -> 命中 ? 304 not modified : 源服务器 -> 无缓存 -> last-modified ? -> 服务器 + if-modified-since + max-age=0 -> 命中 ? 304 not modified : 源服务器 二、cdn缓存 1. cdn(content delivery network),内容分发网络。在用户和源服务的架设的一层服务，它包含负载均衡和高速cache服务的功能。 旨在给用户高速访问、安全的网络环境。 2. 访问原理 无cdn服务请求。 client -> local dns -> origin server IP -> https -> server origin -> content 存在cdn服务请求 client -> local dns -> cdn dns -> cdn 全局负载均衡服务器 -> 区域负载均衡服务器 -> cdn cache服务器 -> origin server 就近原则: cdn全局负载均衡服务器根据用户访问的IP地址和内容，通过内部的算法找到与之对应最近的局域负载服务器 cdn cache服务器：会根据origin server 给出的缓存策略进行缓存，以备下一次用户请求不用直接回源。 回源：cdn cache服务器取不到请求的资源时，会像源服务器发起请求。 CDN缓存什么内容： 缓存html、图片、css、xml、js等静态资源，不缓存含有？的动态地址、jsp、php【除非特殊设置】 缓存原站返回HTTP状态为20*或304，不缓存其他状态(例如404，500，503) 如何刷新cdn 缓存：手动在资源url地址后面添加版本号-> https://www.baidu.com/js/index.js?v=20190331。 通过cdn厂商给出的刷新cdn服务器资源的方法进行刷新。
