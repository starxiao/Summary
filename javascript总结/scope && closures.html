scope && closures 一、作用域 设计一套规则来存储和访问变量，这套规则称为作用域。 1. 编译原理 定义： 程序中的一段代码再执行之前会经历的三个步骤统称为编译 1.1 分词/词法分析，将字符串组成的代码语句提取成若干个代码块组成词法单元。比如var a = 1; 提成 var、a、= 、1、； 1.2 解析/语法分析，将词法单元数组转换成一个由元素逐级嵌套所组成的程序语法树称为抽象语法树（AST）。比如var a = 1; 使用var声明了一个值为1的a变量 1.3 代码生成，将AST转为可执行代码的过程。这个过程包括将AST转成机器指令和分配内存等等操作。 2. 作用域 代码的编译执行过程，有三个工具，负责词法、语法、代码生成的编译器、负责收集和维护变量的作用域、负责程序的编译（转成机器指令）、执行的引擎。 我们将“作用域”定义为一套规则，这套规则用来管理引擎如何在当前作用域以及嵌套的子作用域中根据标识符名称进行变量查找。 作用域链： 从当前作用域一直向上查看，直到全局作用域。 3. 词法作用域 3.1 词法作用域是由你在写代码时将变量和块作用域写在哪里来决定的， 3.2 作用域查找始终从运行时所处的最内部作用域开始，逐级向外或者说向上进行，直到遇见第一个匹配的标识符为止。(由内到外，匹配一个就返回) 3.3 eval 和with会改变词法作用域，但会影响到性能，不要去使用它。 4. 函数作用域和块作用域 4.1 函数作用域的含义是指，属于这个函数的全部变量都可以在整个函数的范围内使用及复用（事实上在嵌套的作用域中也可以使用）。 4.2 隐藏变量因为变量冲突。 4.3 匿名立即执行函数隐藏变量 4.4 块作用域 5. 变量和函数提升 5.1 变量和函数声明从它们在代码中出现的位置被“移动”到了最上面。这个过程就叫作提升 5.2 变量和函数的声明都会在编译阶段进行的，在执行代码之前。 5.3 函数优先被提升，其次才是变量 6. 闭包 6.1 定义： 当函数可以记住和访问它声明时候所在的作用域，即使是在当前作用域之外执行，这时就产生了闭包。 6.2 循环和闭包 6.3 闭包产生模块 模块有两个主要特征：（1）为创建内部作用域而调用了一个包装函数；（2）包装函数的返回 值必须至少包括一个对内部函数的引用，这样就会创建涵盖整个包装函数内部作用域的闭 包。 7. 闭包导致内存泄露 7.1 因为闭包会一直保存函数内的局部变量，导致当函数执行后，该局部变量无法被回收。所以在不使用闭包后，可将闭包置为null
