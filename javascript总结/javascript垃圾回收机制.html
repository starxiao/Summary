javascript 垃圾回收机制 1. 自动垃圾回收机制 找出那些不再使用的变量，然后释放其内存，并且垃圾收集器会按固定时间周期性执行操作 2. 栈内存和堆内存 基本类型在内存中占固定大小的空间，被保存在栈内存中， 引用类型保存在堆内存中。 3. 标记无用变量的策略： 标记清除、引用计数。 标记清除：当变量进入环境时，标记为“进入环境”，变量离开时标记为“离开环境”。收集器发现变量存在标记时，就回收其内存空间。 引用计数：跟踪记录每个值呗引用的次数，当次数为0时，就回收其内存空间。循环引用会导致其不能被回收 4. v8 使用分代回收机制 javascript 内存泄露 1. 全局变量，默认情况下全局变量不会被回收 2. setInterval，在不使用clear掉 3. javascript 闭包 4. dom 节点
