页面渲染过程 一、 页面渲染过程分网络请求过程和浏览器渲染过程 二、网络请求过程 1. url统一资源定位符(Uniform Resource Locator)(www.baidu.com) -> DNS解析成(IP地址) 2. TCP连接三次握手 -> TCP 3. HTTP连接-> 服务返回报文 4. 关闭连接 -> 浏览器解析HTML 5. 文档有资源 -> 直至资源全部加载完成 三、浏览器渲染过程 1. 解析HTML -> HTML DOM树 2. CSS -> CSS DOM树 (解析css过程不会阻塞html的解析, 但会阻塞页面的显示，所以一般会放在head) 3. 执行js(执行js的过程会阻塞文档的解析，因为js可以改变html和css。所以js一般放到body后面) 4. 将HTMLDOM树和CSSDOM树构建成渲染树 5. 布局（回流） -> 从左到右、从上到下进行布局定位,计算每个节点的几何信息 6. 绘制（重绘） -> 浏览器的UI组件将布局过后的渲染树绘制到屏幕上
