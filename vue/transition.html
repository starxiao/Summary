transition组件只能包含一个子节点 在以下情形中，可以给任何元素和组件添加进入/离开过渡 1.条件渲染(v-if) 2.条件展示(v-show) 3.动态组件 4.组件根节点 v-if 跟 v-show 的区别 v-if会重新创建DOM节点。 v-show 不会重新创建DOM节点，通过设置节点的css display实现 v-if 不适合频繁显示与隐藏的操作，频繁操作DOM节点会影响性能 ，v-show不适合大量dom节点树的显示。 transition: v-enter-active v-leave-active v-enter ================> v-enter-to v-leave =================> v-leave-to props: name - string name: 'fade' 类名则为 'fade-enter' enter-class enter-to-class events: before-enter before-leave enter after-enter after-leave leave v-for key的作用 当列表渲染时，如果不使用key时，vue将会最大限度减少动态元素并尽可能的尝试就地修改和复用相同类型的元素。 而使用key时，vue会基于key的变化重新排列元素顺序，并且移除key不存在的元素。 有相同父元素的子元素必须有独特的 key。重复的 key 会造成渲染错误。 它也可以用于强制替换元素/组件而不是重复使用它。当你遇到如下场景时它可能会很有用： 完整地触发组件的生命周期钩子 触发过渡
